# 编译检测功能开发文档

## 1. 需求分析

### 1.1 背景
根据 `编译检测.md` 文档的需求，需要在 IntelliJ MCP 插件中实现一个编译检测功能，用于检查类是否有编译错误或语义错误。

### 1.2 目标
- 实现一个 MCP 工具，能够检查指定类的编译错误
- 支持基于 PSI 的轻量级错误检查
- 返回所有错误消息，而不仅仅是第一条

### 1.3 技术方案
根据 `编译检测.md` 文档，有两种实现方案：
1. **基于 PSI 检查**：使用 IntelliJ 的 PSI (Program Structure Interface) 和 DaemonCodeAnalyzer 获取错误高亮信息
2. **基于编译器检查**：使用 CompilerManager.make 方法触发完整的编译检查

最终选择方案1（基于 PSI 检查），原因：
- 性能更好，响应更快
- 不需要触发完整的编译过程
- 能够检测大部分编译错误和语义错误

## 2. 设计与实现

### 2.1 数据模型设计

#### ClassErrorModels.kt
```kotlin
@Serializable
data class ClassErrorCheckRequest(
    val qualifiedName: String,
    val projectPath: String? = null
)

@Serializable
data class ErrorLocation(
    val message: String,
    val line: Int,
    val column: Int,
    val endLine: Int? = null,
    val endColumn: Int? = null
)

@Serializable
data class ClassErrorInfo(
    val qualifiedName: String,
    val hasErrors: Boolean,
    val errorCount: Int,
    val errorMessages: List<String> = emptyList(),
    val firstErrorMessage: String? = null,
    val checkedByCompile: Boolean = false,
    val errorType: String = "psi",
    val errors: List<ErrorLocation> = emptyList()
)
```

**设计说明**：
- `ClassErrorCheckRequest`：请求参数，包含类的完全限定名
- `ErrorLocation`：单个错误的位置信息
  - `message`：错误消息
  - `line`：错误起始行号（1-based）
  - `column`：错误起始列号（1-based）
  - `endLine`：错误结束行号（可选）
  - `endColumn`：错误结束列号（可选）
- `ClassErrorInfo`：响应结果，包含错误信息
  - `errorMessages`：所有错误消息列表（主要字段）
  - `firstErrorMessage`：第一条错误消息（向后兼容）
  - `errorCount`：错误数量
  - `hasErrors`：是否有错误
  - `errors`：详细错误位置信息列表（新增）

### 2.2 服务层实现

#### ClassErrorCheckService.kt
```kotlin
class ClassErrorCheckService(private val project: Project) {
    fun checkClassErrors(
        request: ClassErrorCheckRequest,
        callback: (ClassErrorInfo) -> Unit
    ) {
        // 1. 查找类
        val psiClass = findPsiClass(request.qualifiedName)
        
        // 2. PSI 错误检查
        val info = checkClassErrorsByPsi(psiClass, request)
        
        // 3. 返回结果
        callback(info)
    }
}
```

**核心实现**：
- `findPsiClass()`：通过 JavaPsiFacade 查找类定义
- `checkClassErrorsByPsi()`：使用 DaemonCodeAnalyzer 获取错误高亮信息
  - 创建临时 Editor
  - 获取 DaemonCodeAnalyzerImpl.getHighlights() 返回的高亮信息
  - 筛选 ERROR 级别的高亮
  - 从 HighlightInfo 提取错误位置信息（行号、列号）
  - 使用 Document.getLineNumber() 将 offset 转换为行号
  - 计算列号：offset - getLineStartOffset(line) + 1

### 3.2 遇到的问题

#### 问题1：CompilerManager.make 方法调用失败
**错误信息**：
```
CompilerManager.make method parameter issues
Return type mismatch
```

**原因**：
- CompilerManager.make 方法的 API 在不同 IntelliJ 版本中有变化
- 回调函数的签名不匹配

**解决方案**：
- 移除编译检查功能，专注于 PSI 检查
- 简化 ClassErrorCheckService，只保留 PSI 检查逻辑
- 更新 ClassErrorCheckRequest，移除 `useCompileMake` 参数

#### 问题2：返回值只包含第一条错误消息
**用户需求**：
需要在工具的返回里面显示所有 error message

**解决方案**：
- 在 `ClassErrorInfo` 中添加 `errorMessages: List<String>` 字段
- 在 `checkClassErrorsByPsi()` 中收集所有错误消息
- 保留 `firstErrorMessage` 字段以便向后兼容

### 3.3 最终实现

#### 修改后的 ClassErrorCheckService.kt
```kotlin
private fun checkClassErrorsByPsi(
    psiClass: PsiClass,
    request: ClassErrorCheckRequest
): ClassErrorInfo {
    // ... 前面的代码 ...

    val errors = mutableListOf<HighlightInfo>()

    ApplicationManager.getApplication().invokeAndWait {
        val editor = EditorFactory.getInstance().createViewer(document, project)
        try {
            val infos = DaemonCodeAnalyzerImpl.getHighlights(
                document, null, project
            )
            val classRange = psiClass.textRange
            for (info in infos) {
                if (info.severity == HighlightSeverity.ERROR) {
                    if (classRange == null || classRange.contains(info.startOffset)) {
                        errors += info
                    }
                }
            }
        } finally {
            EditorFactory.getInstance().releaseEditor(editor)
        }
    }

    return if (errors.isEmpty()) {
        ClassErrorInfo(
            qualifiedName = request.qualifiedName,
            hasErrors = false,
            errorCount = 0,
            errorMessages = emptyList(),
            firstErrorMessage = null,
            checkedByCompile = false,
            errorType = "psi",
            errors = emptyList()
        )
    } else {
        val errorLocations = errors.map { highlightInfo ->
            val line = document.getLineNumber(highlightInfo.startOffset) + 1  // 转换为1-based
            val column = highlightInfo.startOffset - document.getLineStartOffset(line - 1) + 1
            val endLine = document.getLineNumber(highlightInfo.endOffset) + 1
            val endColumn = highlightInfo.endOffset - document.getLineStartOffset(endLine - 1) + 1
            ErrorLocation(
                message = highlightInfo.description,
                line = line,
                column = column,
                endLine = endLine,
                endColumn = endColumn
            )
        }
        val errorMessages = errorLocations.map { it.message }
        ClassErrorInfo(
            qualifiedName = request.qualifiedName,
            hasErrors = true,
            errorCount = errors.size,
            errorMessages = errorMessages,
            firstErrorMessage = errorMessages.first(),
            checkedByCompile = false,
            errorType = "psi",
            errors = errorLocations
        )
    }
}
```

## 7. 技术要点

### 7.1 PSI 检查原理
- PSI (Program Structure Interface) 是 IntelliJ Platform 的核心抽象层
- DaemonCodeAnalyzer 负责后台代码分析，提供错误高亮信息
- HighlightInfo 包含错误的位置、严重程度、描述等信息

### 7.2 异步处理
- 使用 CountDownLatch 实现异步等待
- 设置 30 秒超时，避免长时间阻塞
- 在 EDT (Event Dispatch Thread) 上执行 UI 相关操作

### 7.3 性能优化
- 使用 invokeAndWait 确保在 EDT 上执行
- 及时释放 Editor 资源
- 只检查类范围内的错误，减少不必要的计算
